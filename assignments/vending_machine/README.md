# Vending Machine - RTL & Verification

This repository contains the source code (SystemVerilog), testbenches, and automation scripts for simulating and synthesizing a Vending Machine.

The entire flow for compilation, RTL simulation, logic synthesis, and Gate-Level Simulation (GLS) is fully automated through a robust `Makefile`, utilizing **Synopsys tools (VCS, Verdi, and Design Compiler)**.

---

## 📂 Directory Structure

The project follows a clean architecture, strictly separating the source code files from the artifacts generated by the tools:

* **`rtl/`**: Contains the design files in SystemVerilog (behavioral code).
* **`pkg/`**: Contains package definitions (`.sv`), such as global structs and enums.
* **`sim/`**: Contains the main testbench (`tb_vending.sv`).
* **`synth/`**: Contains the TCL scripts (`synthesize.tcl`, `.synopsys_dc.setup`) and constraints (SDC) for the Design Compiler.
* **`libs/`** *(Auto-generated)*: Directory where the Makefile automatically copies the Verilog and DB libraries from the local PDK (Process Design Kit).
* **`build/`** *(Auto-generated)*: Isolated directory where **all** logs, executables, database files (`.daidir`, `.db`), and waveforms (`.fsdb`) are saved. It is subdivided into `sim_rtl`, `sim_gls`, and `synthesis`.

---

## 🛠️ Prerequisites

To run this project, you must be in a Linux environment configured with access to the following Synopsys tools:

* **VCS / Vlogan** (Compilation and Simulation)
* **Verdi** (Waveform and Schematic Viewer)
* **Design Compiler / Design Vision** (Logic Synthesis and GUI)
* **SAED32 EDK PDK** (Available at `/Tools/PDK/SAED32/...` on the server)

## 🚀 Usage (Makefile Commands)

The `Makefile` is built with a smart dependency chain. You don't need to run it step-by-step manually; simply request the final stage you want to execute or view.

Open the terminal at the project root and use the commands below:

### 1. Full Automated Flow

Executes **everything**: compiles and runs the RTL simulation, performs logic synthesis, compiles the netlist (GLS), and opens the Design Vision GUI with the synthesized schematic.

```bash
make
# or
make all
```

### 2. Behavioral Simulation (RTL)

These commands act on the original pure SystemVerilog code (`rtl/` folder).

* `make rtl_compile` : Only analyzes the syntax and compiles the code.
* `make rtl_run` : Compiles and executes the simulation, generating the log.
* **`make rtl_view`** : Compiles, runs the simulation, and automatically opens **Verdi** to view the waveforms (`waves.fsdb`).

### 3. Logic Synthesis

Converts the RTL code into a Standard Cell *netlist* using the PDK technology library (SAED32nm).

* `make synthesis` : Runs the Design Compiler in the terminal, applying the TCL scripts from the `synth/` folder.
* **`make synth_view`** : Runs the synthesis and opens **Design Vision** to inspect area/timing reports and view the generated physical schematic (*Gate-Level Schematic*).

### 4. Post-Synthesis Simulation (Gate-Level - GLS)

These commands simulate the behavior of the final hardware *netlist* (`vending_machine_syn.v`) alongside the PDK Verilog library in "Zero-Delay". *Warning: If synthesis hasn't been done yet, Make will run it automatically before the GLS compilation.*

* `make gls_compile` : Analyzes the netlist and compiles the GLS simulator.
* `make gls_run` : Executes the GLS simulation.
* **`make gls_view`** : Executes the GLS simulation and opens **Verdi** so you can compare signals and ensure functionality was preserved after synthesis.

## 🧹 Clean-up

Because the project uses an isolated *build* folder, cleaning up generated files is fast and safe, without interfering with the source files.

* `make clean` : Deletes the entire `build/` folder (Simulations and Synthesis). Use this before running a new simulation if you modified module names or fundamental packages.
* `make clean_libs` : Deletes only the copied libraries in the `libs/` folder.
* `make clean_all` : Deletes both `build/` and `libs/`. Returns the repository to its pristine original state (source code only).
